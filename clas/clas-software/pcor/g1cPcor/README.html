g1cPcor Package


  Overview of the Package

This package was written to implement charged particle momentum and tagged photon energy corrections for the g1c run period. Information regarding how the corrections were derived can be found in CLASnote 2004-017. Charged particle corrections, which are performed by the function g1cPcor, are parameterized in terms of |∫Bxdl|/p, θ, φ, charge and sector. To calculate |∫Bxdl|, this package crudely swims the particles through the CLAS magnetic field which is read into memory once when the package is initialized via the InitPcor function. This function also initializes some quantities needed to decode the field map used in calculating |∫Bxdl|. We also make corrections to the charged particle momenta for energy lost in the drift chambers using the g1cDCeloss function. Both charged particle corrections can be made by calling g1cMomCor. Corrections to the tagged photon energy are performed by the function g1cTAGRcor (the tagger corrections can be applied with out calling InitPcor first).

  Compiling and Linking

The InitPcor function reads in the CLAS magnetic field from a .dat file located in this directory. If your CLAS_PACK enviornment variable is set to this packages directory, then the file should be read in correctly. However, if you have CLAS_PACK set to a different version of the packages, one that's old enough not to have this directory, then the code won't know where to find the .dat file. To fix this, either change your CLAS_PACK enviornment variable or just open InitPcor.cc with your favorite editor and put in the path to the .dat file by hand (there are comments near the beginning of the file that tell you where to make the changes).

To compile the library type make lib in this directory. This archives libg1cPcor.a in your TOP_DIR/lib/OS_NAME directory (TOP_DIR and OS_NAME are CLAS enviornment variables). To link to this library you need to add the line -L$(TOP_DIR)/lib/$(OS_NAME) -lg1cPcor to the libs you're linking against. You will also need to add the g1cPcor.h header file to your includes. This can be done by checking g1cPcor.h out of the packages/includes directory (which presumably you're allready linking to), or by adding this directory to your includes with the line -I/????/packages/utilities/g1cPcor replacing the question marks with the appropriate path.

To call these functions in a language other than C++ (i.e. fortran), you have to add the line -lstdc++ (the standard C++ library) to the libs you're linking against (it should come after -lg1cPcor). You'll also have to declare an external prototype of the function prior to calling it (for example, EXTERNAL INITPCOR). In C/C++ you'll, of course, have to include the header file with a #include"g1cPcor.h" statement.

  Applying the Corrections

InitPcor must be called once prior to applying any of the charged particle corrections. g1cMomCor should be run prior to applying the standard ELOSS corections to the particle. The C/C++ version takes the measured 4-momentum as a CLAS vector4_t structure along with the charge and sector as int's and returns the corrected 4-momentum as a CLAS vector4_t structure. The fortran version is a subroutine that takes the measured momentum as a 4-D real array along with the sector and charge as integers and alters the 4-momentum internally. g1cTAGRcor can be called at any time. The C/C++ version takes the measured photon energy, the endpoint energy and a run ID flag (1 for 3.115 g1c, 2 for 2.4 g1c ...those are the only 2 so far, any other flag performs the correction with no beam offset, this actually works very well for g2a) and returns the corrected photon energy. The fortran version is again a subroutine that takes the same arguments and corrects the passed photon energy.
